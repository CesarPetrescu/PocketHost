## Turn 016 - Android CLI and AGP 9 toolchain setup

- Change: installed Android CLI support locally, initialized Android CLI skills for agents, installed Android SDK Platform 36, Build Tools 36.0.0, Platform Tools 37.0.0, Android Emulator 36.5.11, Go 1.26.3, Gradle 9.5.1, and a `medium_phone` Android 36 emulator. Removed the obsolete `org.jetbrains.kotlin.android` plugin entries rejected by AGP 9.
- Blast radius: local build/toolchain setup plus Android Gradle configuration.
- Data flow: Gradle uses `/root/Android/Sdk` from ignored `android/local.properties`; the APK packages daemon artifacts from `android/app/src/main/jniLibs/<abi>/` into the app native library area for `ProcessSupervisor`.
- Verification: `cd go && go test ./...` passed; `bash scripts/package-android.sh` produced `android/app/build/outputs/apk/debug/app-debug.apk`; `android run --apks=android/app/build/outputs/apk/debug/app-debug.apk --device=emulator-5554 --activity=dev.pockethost.MainActivity` installed and launched the app; `dumpsys notification` showed foreground notification id `4401` on channel `pockethost_supervisor`.
- Known gaps: `bash scripts/verify-daemons-local.sh` failed because host port `18080` is already occupied by a `docker-proxy` service. The Android 36 emulator is `x86_64`, while committed PocketHost daemon artifacts are currently only `arm64-v8a`, so Start all reports `Missing binary` on that emulator. Building Android `x86_64` Go daemons requires an Android NDK toolchain; Android CLI did not list NDK packages in this environment, and a direct command-line-tools download stalled.
- Rollback: restore the removed Kotlin Android plugin lines if the project downgrades below AGP 9; remove local SDK/emulator packages if disk usage matters.
- Next improvement: add a Gradle wrapper and either provide emulator-compatible `x86_64` daemon artifacts or document that device verification requires an ARM64 physical device until NDK-backed cross-builds are wired.
