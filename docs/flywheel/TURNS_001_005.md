# Flywheel turns 001-005

This file records five additional PocketHost Flywheel turns applied after the Apache-2.0 finalization pass.

The evidence below is local evidence only. Android APK/device verification and tunnel production verification still require a real device and configured Cloudflare account.

## Turn 001: daemon shutdown is graceful

Change:
- Replaced the basic blocking HTTP server helper with signal/context-aware graceful shutdown.
- Added `NewHTTPServer` and `ServeGracefully` helpers.
- Android `Process.destroy()` now gives Go daemons a chance to drain requests before the process exits.

Blast radius:
- All Go daemons using `pocket.ListenAndServeGracefully`: `hostd`, `webd`, `filed`, `proxyd`, `ddnsd`.

Data flow:
- Signal or context cancellation -> HTTP `Shutdown()` -> listener closes -> daemon exits cleanly.

Tests run:
- `go test ./...`
- Added `TestServeGracefullyStopsOnContextCancel`.

Device tested:
- Not in this environment.

Known gaps:
- Needs Android Stop All validation with real process lifecycle evidence.

Rollback:
- Revert `go/internal/pocket/common.go` to the previous `http.Server.Serve` implementation.

## Turn 002: MiniCloud rejects symlink escape paths

Change:
- Added `SafeExistingPath` for existing file operations.
- Replaced generic `http.FileServer` download path with explicit safe download handling.
- Disabled directory listing on `/files/`; listing stays behind `/api/files`.
- Upload now refuses to overwrite existing symlinks.

Blast radius:
- `filed` list/download/upload/delete operations.

Data flow:
- Client path -> `SafeJoin` -> existing-path check -> symlink-component rejection -> filesystem operation.

Tests run:
- `go test ./...`
- Added regression tests for symlink download escape, symlink-parent delete escape, directory listing, token behavior, and upload symlink overwrite.

Device tested:
- Not in this environment.

Known gaps:
- Share links are not implemented yet; they need the same path-safety policy before exposure.

Rollback:
- Revert `go/cmd/filed/main.go` and `go/cmd/filed/main_test.go`.

## Turn 003: reverse proxy routes are validated before use

Change:
- Added route-host normalization.
- Added target URL validation.
- Dropped invalid route entries instead of accepting malformed schemes/hosts.
- Rejected unspecified targets such as `0.0.0.0` and `[::]`.

Blast radius:
- `proxyd` route parsing and host routing.

Data flow:
- Config string -> parse -> normalize host -> validate target URL -> route map -> proxy director.

Tests run:
- `go test ./...`
- Added invalid-route tests and target URL tests.

Device tested:
- Not in this environment.

Known gaps:
- Later UI route editor should surface rejected routes instead of silently ignoring them.

Rollback:
- Revert `go/cmd/proxyd/main.go` and `go/cmd/proxyd/main_test.go`.

## Turn 004: Android supervisor distinguishes process-alive from health-ok

Change:
- Added Kotlin `HealthMonitor`.
- Added `ServiceState.Degraded`.
- `ProcessSupervisor` now probes `http://127.0.0.1:<port>/health` after daemon start and marks a service degraded after repeated failures.
- Dashboard attention count includes degraded services.

Blast radius:
- Android supervisor status model and UI state display.

Data flow:
- Daemon process starts -> foreground supervisor polls local `/health` -> status map updates -> Compose UI renders health state.

Tests run:
- Static/local checks only. Android build was not available in this environment.

Device tested:
- Not in this environment.

Known gaps:
- Needs Android Studio/Gradle compile and on-device probe evidence.
- Health probe currently assumes `/health` for all port-bearing services; Matrix may need `_matrix/client/versions` once the Matrix binary is selected.

Rollback:
- Remove `HealthMonitor.kt`, remove `Degraded`, and remove `launchHealthLoop` from `ProcessSupervisor`.

## Turn 005: CI now proves daemon health, not just compilation

Change:
- Added `scripts/verify-daemons-local.sh`.
- CI now builds local daemon binaries, starts them on high local ports, probes `/health`, verifies a proxyd host route, and verifies the filed list API.
- Added Makefile `verify-daemons` target.
- Rebuilt Android ARM64 Go daemon artifacts after the code changes.

Blast radius:
- Local CI/runtime verification.

Data flow:
- `go build` -> daemon process -> local HTTP health endpoint -> Python verifier -> CI result.

Tests run:
- `./scripts/ci-local.sh`
- `./scripts/build-go-android.sh arm64-v8a`
