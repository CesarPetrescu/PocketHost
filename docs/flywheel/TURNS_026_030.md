# Flywheel turns 026-030

## Turn: web control panel + manager redesign + multi-ABI build

Change:
- Added a loopback web control panel served by `hostd` (embedded SPA + gateway:
  `/api/services` aggregation and token-forwarding proxies to ddnsd/filed).
- Redesigned the in-app Compose manager (Material 3 theme, status chips, uptime,
  state-aware controls, web-panel launcher).
- Added an explicit, off-by-default network exposure toggle (loopback ⇄ 0.0.0.0)
  wired through `ServiceRegistry` and `POCKETHOST_ALLOW_PUBLIC_BIND`.
- Made the app build and install across architectures: fixed `armeabi-v7a` in
  `build-go-android.sh` (android/arm needs cgo), rebuilt all four ABIs, added the
  Gradle wrapper, ABI splits, and a debug-signed release build type.
- Fixed in-app health probes failing under Android's cleartext policy by adding a
  loopback-only network security config.

Blast radius: `hostd` (new routes; existing routes unchanged), Android UI and
supervisor preferences, the daemon build script, and packaged `.so` artifacts.
No change to default bindings (still loopback) or to the loopback bind guard.

Data flow: see docs/DATA_FLOW.md "Web control panel flow". Browser → hostd with
admin token; hostd re-authenticates outbound to siblings so the token stays on
loopback. Start/stop remains exclusively with the Android supervisor.

Tests run:
- `go test ./...` (incl. new `gateway_test.go`) — pass, uncached.
- `gofmt -l` clean; `go vet ./...` clean.
- `./scripts/ci-local.sh` — pass, incl. new checks: panel served, `/api/services`
  token-gated and aggregating 6 daemons.
- `gradle :app:compileDebugKotlin` and `:app:assembleDebug` — success; per-ABI +
  universal APKs produced; x86_64 split verified to contain only x86_64 daemons.

Device tested: headless `x86_64` emulator (API 36, KVM).

Daemon health evidence: after Start all, child processes `libhostd/webd/filed/
proxyd.so` run under the app uid; on-device `/health` returns `"status":"ok"` for
8099/8080/8090/8088; ddnsd correctly not started (off by default). After enabling
the exposure toggle and "Restart to apply", `hostd` reports `addr 0.0.0.0:8099`
and `ss -tlnp` shows all four ports on `*`.

Logs/screenshots: docs/screenshots/ (01–09), including the web panel running in
mobile Chrome with a live status grid.

Known gaps:
- Release APK is debug-signed (no production keystore / Play pipeline — gated).
- No automated Android instrumented test yet; emulator smoke was manual.
- Matrix slot still empty; cloudflared tunnel import/route UI still pending.
- Code/resource shrinking left off until daemon-launch keep rules are verified.

Rollback: revert the listed files; the previous loopback-only ARM64/x86 build and
