# Flywheel turns 006-015

These turns continue from `TURNS_001_005.md`.

## Turn 006 - shared auth and HTTP middleware

- Change: added constant-time token checking, strict bearer parsing, security headers, structured request logging, and JSON error helpers in `go/internal/pocket`.
- Blast radius: all Go HTTP daemons that use `pocket.NewHTTPServer`, `RequireToken`, or `RequestLog`.
- Data flow: request -> security headers -> optional token gate -> daemon handler -> structured log with request id/status/bytes.
- Verification: `TestConstantTimeTokenEqual`, `TestBearerTokenRejectsMalformedAuthorization`, `TestRequestLogRecordsStatusAndRequestID`.
- Rollback: revert to previous `RequireToken`; keep public-bind and file-server hardening independent.

## Turn 007 - loopback-only bind guard

- Change: added `ValidateListenAddr` and `ResolveListenAddr`; daemons now reject non-loopback bindings unless `POCKETHOST_ALLOW_PUBLIC_BIND=true` is explicitly set.
- Blast radius: daemon startup; public exposure becomes opt-in.
- Data flow: CLI/env addr -> `ResolveListenAddr` -> `net.Listen`.
- Verification: `TestValidateListenAddrDefaultsToLoopback` and live verification that `webd --addr 0.0.0.0:18180` fails by default.
- Rollback: set `POCKETHOST_ALLOW_PUBLIC_BIND=true` for a deliberate LAN exposure test.

## Turn 008 - safe `webd` static serving

- Change: replaced `http.FileServer` with an explicit static handler using `SafeExistingPath`; directory listing is disabled and symlink escapes are rejected.
- Blast radius: web static serving only.
- Data flow: URL path -> clean relative path -> symlink-aware path check -> file serve.
- Verification: `TestStaticHandlerRejectsTraversal`, `TestStaticHandlerRejectsSymlinkEscape`, `TestStaticHandlerDisablesDirectoryListing`, and live directory-listing check.
- Rollback: none recommended; this closes a class of file exposure bugs.

## Turn 009 - bounded and atomic MiniCloud uploads

- Change: `filed` uploads now use configurable `--max-upload-bytes` / `POCKETHOST_FILED_MAX_UPLOAD_BYTES`, `http.MaxBytesReader`, temporary files, chmod, and atomic rename.
- Blast radius: upload endpoint.
- Data flow: multipart body -> byte cap -> temp file inside target directory -> chmod -> rename to final path.
- Verification: `TestUploadRespectsMaxBytes`, `TestUploadCommitsAtomically`, existing symlink-overwrite regression tests.
- Rollback: raise the cap if real devices need larger uploads; do not remove the cap.

## Turn 010 - DDNS validation and update discipline

- Change: DDNS now validates public IP responses, restricts record type to A/AAAA, validates IP version, supports injectable IP/API URLs, and skips Cloudflare patch calls when the IP is unchanged after a successful update.
- Blast radius: `ddnsd` only.
- Data flow: IP endpoint -> public-IP validator -> record-type validator -> Cloudflare PATCH.
- Verification: `TestPublicIPFromRejectsPrivateIP`, `TestValidateRecordTypeMatchesIPVersion`, `TestUpdateOncePatchesCloudflare`, `TestUpdateOnceSkipsUnchangedIPAfterSuccess`.
- Rollback: keep validation; adjust endpoint or record type config instead.

## Turn 011 - token gates for host and DDNS admin APIs

- Change: `/api/status` on `hostd` and `/api/update-now` on `ddnsd` now require `POCKETHOST_TOKEN`; `/health` stays unauthenticated for local probes.
- Blast radius: local admin API callers.
- Data flow: Android-generated admin token -> daemon environment -> token middleware -> admin endpoint.
- Verification: `TestHostStatusRequiresTokenButHealthDoesNot`, `TestDDNSHandlerRequiresTokenForUpdate`, live hostd/filed token checks.
- Rollback: pass the token from Android or disable the token intentionally only in throwaway dev mode.

## Turn 012 - proxy runtime hardening

- Change: `proxyd` now compiles routes once, logs dropped malformed routes, uses a bounded `http.Transport`, sets an explicit 502 error response for upstream failures, and adds an upstream evidence header.
- Blast radius: reverse proxy routing.
- Data flow: Host header -> normalized route -> precompiled reverse proxy -> bounded upstream transport.
- Verification: `TestProxyForwardsHostAndAddsUpstreamHeader`, `TestProxyReturnsBadGatewayForUpstreamError`, existing route-validation tests, live `web.local -> webd` check.
- Rollback: restore old request-time proxy creation only if debugging a route compilation issue.

## Turn 013 - Android log redaction

- Change: added `SecretRedactor`; `LogBus.emit` now redacts admin tokens, bearer tokens, and common token/secret/password assignment patterns before storing/displaying logs.
- Blast radius: Android log display and SQLite log storage.
- Data flow: daemon stdout -> `ProcessSupervisor` -> `LogBus.emit` -> redaction -> SQLite/UI.
- Verification: source review and CI-preserved Kotlin source; Android runtime verification still required.
- Rollback: narrow regexes if they over-redact useful diagnostics; do not store secrets unredacted.

## Turn 014 - Android log retention

- Change: SQLite log persistence now prunes to the newest 5,000 rows after inserts.
- Blast radius: Android diagnostics/log history.
- Data flow: log line -> SQLite insert -> retention prune.
- Verification: source review; Android runtime verification still required.
- Rollback: adjust retention count if device storage or support needs differ.

## Turn 015 - diagnostics bundle and stronger local CI

- Change: added Android diagnostics zip creation from the Settings screen; local CI now checks Go formatting and live daemon verification now proves bind guard, token gates, security headers, proxy route, and directory-listing block.
- Blast radius: diagnostics UX and CI.
- Data flow: Settings -> `Diagnostics.createBundle` -> zip manifest/logs/database in app-private data.
- Verification: `./scripts/ci-local.sh` passed after changes; Android UI click still requires device verification.
- Rollback: diagnostics feature is additive and can be hidden without affecting daemon runtime.

## Known gaps after turn 015

- Android APK build still requires Android SDK/Gradle outside this container.
- Android runtime behavior still needs device evidence: foreground notification, daemon launch, stop behavior, boot autostart, diagnostics bundle creation, and battery/thermal observations.
- Matrix homeserver binary is still a slot, not a bundled implementation.
- Cloudflare Tunnel still requires a user-supplied official `cloudflared` binary/config and real tunnel verification.
